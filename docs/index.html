Features

Hands-free Discord voice

A sidecar that connects Discord voice channels to voice-echo's full audio pipeline. No changes to voice-echo required.

👥

Auto Join/Leave

Bot joins when users enter the voice channel, leaves when empty. No manual commands needed.

presence events
🔄

Codec Bridge

Handles all conversion: Opus to mu-law, 48kHz to 8kHz, and back. voice-echo sees standard audio, Discord gets standard audio.

opus ↔ mu-law
🔁

Auto-Reconnect

If voice-echo restarts, reconnects with exponential backoff from 1s to 30s. No manual intervention.

exp. backoff
👤

SSRC Mapping

Maps Discord audio streams to user IDs for multi-user identification. Know who's talking in a group channel.

multi-user
🔒

DAVE E2EE

Discord's end-to-end encryption support via a custom songbird fork. Voice data stays encrypted in transit.

encryption

Architecture

Separate binary, intentionally

Why a sidecar instead of a feature flag inside voice-echo?

Songbird pulls ~40 crates. Users who only want phone calls don't pay this cost. discord-voice-echo is a standalone binary that talks to voice-echo over the same interface Twilio uses -- keeping both projects lean and independent.

Quick start

Up and running in minutes

Clone, configure, run. You'll need a Discord bot token and a running voice-echo instance.

1

Clone and build

git clone https://github.com/dnacenta/discord-voice-echo.git cd discord-voice-echo cargo build --release
2

Configure

mkdir -p ~/.discord-voice-echo # Create config with your bot token and IDs cat > ~/.discord-voice-echo/config.toml << EOF discord_token = "your-bot-token" guild_id = "your-guild-id" channel_id = "your-voice-channel-id" EOF
3

Run

./target/release/discord-voice-echo
4

Join a voice channel and talk

# The bot auto-joins when you enter the voice channel # Just start talking -- Echo is listening
